FASE 4 COMPLETADA: CIERRE DIARIO CON FIRESTORE + FALLBACK LOCAL

Qué quedó listo:
- cierre-diario.service.ts ahora usa el SDK 'firebase' oficial
- No depende de @angular/fire
- Si Firestore está disponible en runtime, sincroniza con la colección: cierresDiarios
- Si Firestore no está disponible o falla, usa localStorage sin romper la UI
- Se creó el modelo: src/app/features/cierre-diario/models/cierre-diario.model.ts
- Se integra con ChecklistService para calcular automáticamente métricas del checklist del día

Métodos principales disponibles:
- listarReportes()
- obtenerReportes()
- listarCierres()
- obtenerReportePorFecha(fecha)
- ultimoReporte()
- generarReporte(usuario?)
- generarCierre(usuario?)
- cerrarDia(usuario?)
- actualizarReporte(id, changes)
- eliminarReporte(id)
- reemplazarDesdeRespaldo(items)
- exportarReporte(fileName, content)
- sincronizarLocalAFirestore()

Sugerencia de integración:
1) generar reporte del día:
   const reporte = this.cierreDiarioService.generarReporte(this.authService.usernameActual() || 'Sistema');

2) exportar reporte:
   this.cierreDiarioService.exportarReporte('cierre-dia.json', JSON.stringify(reporte, null, 2));

3) listar historial:
   const reportes = this.cierreDiarioService.listarReportes();
